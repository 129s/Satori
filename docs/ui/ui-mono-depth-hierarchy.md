# UI 调整建议（单色系 / 不换字体 / 保留现有旋钮方案）

目标：在“精密仪器感”与“物理质感”之间取得平衡，解决扁平化过度、缺层级、比例失调与信息组织弱的问题。配色保持单色系（现有 Cyan/Teal 主色），字体保持现状，旋钮暂不重做。

## 1. 布局与背景（Depth & Hierarchy）

- 全局背景：用深灰而非纯黑（建议 `#1E1E1E`）
- 模块卡片：Excitation / String / Body / Room 各自独立背景板
  - 背景：`#2C2C2C`
  - 圆角：`6px`
  - 微弱阴影：`0 4px 6px rgba(0,0,0,0.3)`（或等效的 D2D 阴影/外描）
- 间距：
  - 外边距：界面边缘至少 `16px`
  - 模块间距：`10px`
  - 模块内边距：`12~16px`

## 2. 可视化区域（Top Row）

原则：从“Debug 线条”变成“仪器反馈”，减少噪点、提升层次，但保持信息密度克制。

- 统一高度：例如固定 `120px`
- 线条：开启抗锯齿，线宽 `1~1.5px`
- 填充：在曲线下方加 0.15~0.25 不透明度的同色系渐变填充（上实下虚）
- 激活态：仅提高饱和度/亮度与轻微 glow，避免额外颜色引入

各模块建议：

- Excitation：显示激励波形/瞬态包络；可用渐变填充增加体积感
- String：由“粗柱”改为“细线频谱/谐波列”（30~40 根 1px 竖线或简化谐波曲线）
- Body：用频响/共鸣“山峰曲线”替代无意义的实心块（Tone 左右移动，Size 变宽变窄）
- Room：保持面板信息结构清晰（选择/波形/控制），选中态可用同色系微弱发光

## 3. 文本与层级（不换字体）

- 模块标题（SECTION）：`10px`、全大写、`letter-spacing: 1.2~1.5px`、颜色 `#8A8A8A`
- 参数标签（Label）：`11px`、颜色 `#CFCFCF`、统一放在控件下方
- 数值提示：尽量“就地显示”（控件附近），减少飘忽的 Tooltip 遮挡

## 4. 单色系配色（Monochrome）

- 主色（Accent）：沿用现有 Cyan/Teal，但降低刺眼度（例如 `#00D8FF` 级别）
- 状态区分：用饱和度/明度区分，不引入第二色
  - 激活：更亮、更饱和 + 轻微 glow
  - 未激活：降低饱和度/亮度
  - 禁用：进一步降低对比度
